## Usage Notes

**Primary Use Cases**:
1. Search for architectural best practices relevant to this TypeScript/Node.js project
2. Find design pattern references for clean architecture implementation
3. Discover testing strategies for domain-driven systems
4. Identify performance optimization techniques for vector search systems
5. Locate error handling patterns for Node.js applications

**Search Strategy**:
- Use `concept_search` for high-precision searches on specific concepts (e.g., "repository pattern")
- Use `broad_chunks_search` for comprehensive questions (e.g., "How to implement clean architecture in TypeScript?")
- Use `catalog_search` to discover relevant architectural documentation

**Expected Knowledge Base**:
- Software architecture books (e.g., Clean Architecture, DDD)
- TypeScript/Node.js best practices documentation
- Design pattern references
- Vector database and search system documentation
- Software engineering principles and practices
